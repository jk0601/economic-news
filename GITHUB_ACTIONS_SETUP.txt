================================================================================
          GitHub Actions (Cron) 으로 경제 뉴스 봇 실행하기
================================================================================

■ 준비
--------------------------------------------------------------------------------
  1) 이 저장소를 GitHub 에 푸시합니다.
  2) .env 는 .gitignore 되어 있어 올라가지 않습니다. (로컬 키는 .env 에 두는 것을 권장)
  3) 저장소에 아래 "Repository secrets" 를 등록합니다.

  경로: GitHub 저장소 → Settings → Secrets and variables → Actions → New repository secret


■ 등록할 Secret 이름 (이름을 정확히 맞추세요)
--------------------------------------------------------------------------------
  NAVER_CLIENT_ID
  NAVER_CLIENT_SECRET
  NEWSAPI_KEY
  OPENAI_API_KEY
  NOTION_TOKEN
  NOTION_DATABASE_ID

  값은 apikey_how.md 와 동일하게 발급받은 키·토큰·DB ID 를 그대로 넣습니다.


■ 워크플로 파일
--------------------------------------------------------------------------------
  .github/workflows/economic-news-cron.yml
    - workflow_dispatch: Actions 탭에서 수동 실행
    - schedule: 매일 UTC 00:00 (한국 시간 09:00)


■ Cron 시간 바꾸기
--------------------------------------------------------------------------------
  economic-news-cron.yml 의 schedule.cron 을 수정합니다.
  반드시 UTC 기준입니다.

    한국 KST = UTC + 9
    예: 한국 매일 오후 6시 → UTC 오전 9시 → cron: "0 9 * * *"


■ 로컬 개발 (기존 방법 유지)
--------------------------------------------------------------------------------
  방법 A) .env 파일
    - .env.example 을 복사해 .env 로 저장
    - 각 변수에 키 입력
    - pip install -r requirements.txt 후 python main.py

  방법 B) config.py 문자열 수정
    - 저장소의 config.py 에 YOUR_* 자리에 직접 넣을 수 있으나, 실수로 push 하면 키가 노출됩니다.
    - 가능하면 방법 A(.env)만 사용하세요.


■ 주의
--------------------------------------------------------------------------------
  - scheduled 워크플로는 기본 브랜치(main 등)의 최신 워크플로 정의를 사용합니다.
  - 무료 계정의 Actions 분이 소진되면 실행이 막힐 수 있습니다.
  - 뉴스/API/Notion 호출이 매 실행마다 발생하므로 cron 간격은 필요에 맞게 조정하세요.
